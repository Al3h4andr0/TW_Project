Introducere

Scop

Aceasta documentatie vizeaza aplicatia web EMa (Estate Web Manager) care gestioneaza tranzactiile imobiliare.

Audienta Vizata si Lecturi Sugerate

Acest document este destinat developerilor care au o baza bine definita in html, css, javascript.

Este recomandat sa se citeasca mai intai sectiunile Prezentare Generala, Caracteristicile Sistemului apoi restul.

Domeniul de Aplicare

EMa manageriaza eficient tranzactiile imobiliare prin a oferi posibilitatea oricarei persoane sa puna in chirie sau sa inchirieze un imobil cu ajutorul unui sistem cartografic interactiv cu filtre relevante. Intr-un cadru antreprenorial cei care dau in chirie pot folosi serviciile oferite de aplicatie pentru o taxa procentuala. Pentru a mentine relevanta aplicatia se pot adauga feature-uri noi in functie de piata imobiliara actuala.

Prezentare Generala

Perspectiva Produsului

Aplicatia este un produs nou, de sine statator care are in componenta sa Leaflet - serviciu de cartografie avand o interfata grafica proprie.

Functionalitatile Produsului

Functionalitatile de baza ale aplicatiei sunt:

  • managementul tranzactiilor imobiliare folosind chestionare cu date personale, conturi pe aplicatie

  • cautarea interactiva a imobilelor folosind harta, taguri, bara de cautare, review-uri

Caracteristici si Utilizatori

Persoanele care vor folosi aplicatia sunt oameni de rand, majoritatea adulti care cauta un loc de cazare (turisti) si antreprenori care vor pune la dispozitie imobilurile. Nu este necesar pentru turisti sa aiba cunostiinte speciale pentru a folosi aplicatia, in schimb antreprenorii vor avea nevoie de cunostiinte de business pentru a colabora cu detinatorii aplicatiei.

Constrangeri ale Design-ului si ale Implementarii

Aplicatia foloseste exclusiv resurse cu licenta open source. Nu exista limitari hardware. Foloseste serviciul Leaflet impreuna cu interfata sa . Ca tehnologii folosite avem geolocatia, incriptia care este folosita pentru date personale folosinf JWT , modelul MVC.

Tool-urile folosite pentru developementul aplicatiei sunt:

  • Editor text: Visual Studio Code

  • Validatoare: W3schools html Validator, CSS Portal Validator, web.dev Quality Validator

  • Management al codului: Github

Mediul de Operare

Aplicatia va rula exclusiv pe orice browser web popular(Ex. Chrome, Firefox, Edge, Safari), specificatiile hardware fiind cele minime pentru navigarea acestor platforme. Este independent de orice sistem de operare dar developerii au dezvoltat o pe Windows 10.

Documentatie Pentru Utilizator

Documentatia pentru utilizator este disponibila pe pagina principala a proiectului sauaici , alt mod de prezentare/explicare a aplicatiei este videoul explicativ care este trimis impreuna cu proiectul.

Ipoteze si Dependente

Dependentele aplicatiei sunt serviciile Leaflet.

Cerinte Externe pentru Interfata

Interfata Utilizatorului

Pagina principala este formata din harta oferita de Leaflet, iar pe deasupra exista diferite butoane cu ajutorul carora utilizatorul poate vizualiza diferite imobile in functie de preferinte pe harta (locatia) iar in stanga o descriere urmata de multiple detalii despre imobil.

Utilizatorul are optiunea si de a cauta un imobil, cu ajutorul field-ului search din bara de navigare situata in antetul paginii. De asemenea, in bara de navigare exista un buton cu ajutorul caruia utilizatorul isi poate adauga un imobil spre vanzare/inchiriere dar numai cu conditia ca acesta sa fie logat.

Logarea se poate realiza cu butonul din dreapta barei de navigare, buton care deschide un pop-up unde se poate realiza logarea, sau ( daca logarea a fost realizata ) modificarea/vizualizarea datelor contului si lista imobilelor postate de acest user.

Dupa logare, utilizatorul poate accesa functia de add prin intermediul butonului specificat mai devreme, buton care va trimite utilizatorul intr-un formular separat in care va introduce datele necesare pentru a adauga imobilul, inclusiv fotografii.

Interfata Hardware

Din moment ce aceasta aplicatie va fi incarcata pe web, host-ul ales pentru produsul finit este cel oferit de platforma Heroku .

Aplicatia noastra web poate fi accesata de pe multiple dispozitive, aceasta fiind facila pentru orice device care are la dispozitie un browser actualizat.

Interfata Software

Aplicatia va fi creata in HTML5, CSS3, JavaScript ES6 pe partea de UI, iar pentru gestionarea imobilelor si a conturilor vom folosi fisiere cu extensia .json.

Interfata de Comunicare

Orice utilizator cu o conexiune la internet si cu ajutorul unui browser poate accesa aplicatia prin protocol HTTP. Va exista si o rubrica de suport prin care utilizatorul poate semnala anumite erori, imbunatatiri etc.

Caracteristicile Sistemului

Pagina de Cautare - Rezultate

In partea stanga a paginii, va exista o zona in care imobilele pot fi vizualizate sub forma de lista. Lista se va actualiza in functie de input-ul utilizatorului in bara de search respectiv in functie de filtrele aplicate.

In aceasta lista, sunt afisate doar informatii esentiale:

  • O poza reprezentativa

  • Pretul cerut

  • Zona/Strada

In momentul in care utilizatorul apasa pe o proprietate, partea stanga a site-ului va fi populata cu informatii in detaliu despre acea proprietate:

  • Overview-ul imobilului

  • Review-urile imobilului (In cazul in care e dat spre inchiriere)

  • Pozele de la locul respectiv

  • Tag-urile asociate

  • Buton pentru a sugera posibilitatea de a inchiria/cumpara imobilul

Inputul Utilizatorului

Partea de stanga sus a paginii web va fi populata cu logo-ul aplicatiei, o bara de search si o zona in care se pot adauga filtre si/sau ordonari precum:

  • Pret intre 1000-3000 lei

  • Zona: Copou si Centru

  • Sortat dupa pret

  • Theft rate/month

Cuvintele cheie inserate in bara de search, respectiv filtrele aplicate de catre utilizator vor avea impact in zona de rezultate, unde imobilele vor avea cerintele dorite de catre utilizator.

Administrarea Identitatii Utilizatorului

In partea dreapta sus a paginii, exista un buton ce va facilita utilizatorului optiunea de a se loga cu un ID si parola sau a-si vizualiza contul. Aceste optiuni sunt vizibile intr-o fereastra de tip pop-up. In aceasta fereastra, se pot schimba datele personale precum numar de telefon etc sau pentru a se vizualiza imobilele postate de catre utilizator. Daca user-ul nu are un cont, va avea optiunea de a-si crea unul, urmand ca apoi sa aiba optiunea de a adauga un imobil pe platforma.

Daca user-ul are imobile pe platorma, se pot vizualiza din aceasta zona, iar daca se incearca accesarea lor prin click, se vor deschide si vor putea fi vizualizate in zona Pagina de Cautare - Rezultate, cu butoane aditionale pentru shimbarea detaliilor imobilului.

Leaflet

Partea neocupata de catre Pagina de Cautare - Rezultate si Inputul Utilizatorului va fi populata de catre API-ul Leaflet, unde vor fi waypoint-uri actualizate in functie de input-ul utilizatorului. Waypoint-urile pot fi accesate, iar daca acest lucru se intampla, in partea de Building Description se vor afisa informatiile imobilului reprezentat de acel waypoint. Evident, se pot folosi si functionalitatile oferite din oficiu de catre Leaflet, precum zoom in/out, navigare prin swipe etc.

Adaugarea unui Imobil

In partea dreapta sus a paginii, langa butonul ce se ocupa de managerierea contului, exista un buton de Add pe care utilizatorul il va putea apasa pentru a lista o locuinta noua.

Acest buton il va trimite pe o alta pagina, un formular, unde vom colecta informatii despre noul imobil ce va fi listat. Aceste informatii sunt (in linii mari) urmatoarele:

  1. Date despre proprietar

  2. Fotografii cu interiorul/exteriorul imobilului

  3. Adresa imobilului

  4. Pret

  5. Daca imobilul este dat spre inchiriere sau vanzare

  6. Filtre despre locatie ( faciliatati, temp. anuala etc.)

  7. O scurta descriere a imobilului.

Formularul Pentru Crearea unui Cont Nou

Dupa cum am spus si in sectiunea 'Administrarea Identitatii Utilizatorului' ( click aici ) exista un buton in partea dreapta sus a paginii care va deschide un pop-up unde utilizatorul se poate loga iar, daca acesta nu are un cont, are optiunea si de a-si crea unul.

Optiunea de a crea un cont este data de butonul 'Register', buton ce va redirectiona utilizatorul spre formularul pregatit pentru a colecta datele necesare crearii contului.

Datele esentiale crearii unui cont sunt:

  • Nume

  • Prenume

  • Adresa eMail

  • Parola

Dupa ce toate aceste date sunt introduse, utilizatorul va apasa pe butonul 'Register' ce il va trimite inapoi la pagina principala unde se va putea loga.

Alte Cerinte Non-Functionale

Cerinte Legate de Performanta

Aceasta aplicatie web nu are componente ce au nevoie de o putere mare de procesare, astfel incat, orice device cu un browser instalat cu o versiune nu mai veche de cativa ani ani o poate folosi.

Cerinte Legate de Siguranta

Se va implementa o metodologie ce va opri atacuri de tip SQL injections, iar atacurile de tip JS injection vor fi gestionate automat.

Cerinte Legate de Securitate

Datele confidentiale ale utilizatorului vor fi trimise in mod encriptat catre server si inapoi, alte date ce vor fi sustrase in mod automat sunt doar fisierele de tip imagine.

Atribute ce tin de Calitatea Partii de Software

Din moment ce aplicatia este construita pe o arhitectura de tip microservicii, aceasta poate fi modificata cu usurinta. De asemenea, designul este unul responsive pentru compatibilitate sporita pe nenumarate platforme.
